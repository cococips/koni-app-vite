[INTRO]

[BAGIAN 1 - AUTENTIKASI JWT]
Sistem autentikasi kami menggunakan JSON Web Token atau JWT.
Ketika user login, backend memvalidasi username dan password,
lalu menghasilkan token yang berisi informasi role pengguna.

  router.post('/login', async (req, res, next) => {
    const { username, password } = req.body

    // Cari user di database
    const [rows] = await db.query(
      'SELECT * FROM users WHERE username = ?', [username]
    )
    const user = rows[0]

    // Verifikasi password dengan bcrypt
    const match = await bcrypt.compare(password, user.password_hash)
    if (!match) return res.status(401).json({ message: 'Password salah.' })

    // Generate JWT token
    const payload = { id: user.id, username: user.username,
                      role: user.role, ref_id: user.ref_id }
    const token = jwt.sign(payload, process.env.JWT_SECRET, { expiresIn: '7d' })

    res.json({ success: true, token, user: payload })
  })

Ada beberapa poin penting di sini. Password tidak disimpan
sebagai plain text — kami menggunakan bcrypt untuk hashing
yang sangat aman. JWT berisi payload role (admin/pelatih/atlet/
wasit) sehingga backend tahu siapa yang mengirim request.
Token berlaku selama 7 hari sebelum expired.

Yang membuat sistem ini kuat adalah satu endpoint login untuk
semua role — sistem otomatis mendeteksi role dari database dan
mengarahkan ke dashboard yang tepat.

[BAGIAN 2 - MIDDLEWARE AUTH & ROLE GUARD]
Agar route backend terlindungi, kami membuat dua middleware:

  // Verifikasi token
  function authenticate(req, res, next) {
    const token = req.headers.authorization?.split(' ')[1]
    if (!token) return res.status(401).json({ message: 'Token tidak ada.' })

    try {
      req.user = jwt.verify(token, process.env.JWT_SECRET)
      next()
    } catch {
      return res.status(401).json({ message: 'Token tidak valid.' })
    }
  }

  // Guard berdasarkan role
  function authorize(...roles) {
    return (req, res, next) => {
      if (!roles.includes(req.user.role)) {
        return res.status(403).json({ message: 'Akses ditolak.' })
      }
      next()
    }
  }

  // Penggunaan di route:
  const guard = [authenticate, authorize('pelatih')]
  router.get('/me', ...guard, async (req, res) => { ... })

Dengan middleware ini, route /api/pelatih/me hanya bisa
diakses oleh user dengan role 'pelatih' yang memiliki token
valid. Kalau admin mencoba akses, langsung ditolak 403.

[BAGIAN 3 - AXIOS INSTANCE DI FRONTEND]
Di sisi frontend, kami membuat Axios instance terpusat di
file src/api/axios.js:

  import axios from 'axios'

  const api = axios.create({
    baseURL: 'http://localhost:5000/api',
  })

  // Interceptor — otomatis sisipkan token ke setiap request
  api.interceptors.request.use(config => {
    const token = localStorage.getItem('koni_token')
    if (token) config.headers.Authorization = `Bearer ${token}`
    return config
  })

  // Interceptor response — kalau 401, redirect ke login
  api.interceptors.response.use(
    res => res,
    err => {
      if (err.response?.status === 401) {
        localStorage.removeItem('koni_token')
        window.location.href = '/login'
      }
      return Promise.reject(err)
    }
  )

  export default api

Keuntungan pola ini: token tidak perlu ditulis manual di setiap
request. Cukup import api dan langsung pakai — token otomatis
tersisipkan. Kalau token expired, user otomatis di-logout.

[BAGIAN 2 - CRUD ATLET VIA API]
Sekarang kita lihat bagaimana CRUD bekerja di AtletPage.
Berbeda dari sebelumnya yang pakai localStorage, sekarang
semua data diambil dan disimpan langsung ke MySQL melalui API.

  export default function AtletPage() {
    const [atlet, setAtlet] = useState([])

    // Load data dari API saat komponen mount
    useEffect(() => { load() }, [])

    async function load() {
      const r = await api.get('/admin/atlet')
      setAtlet(r.data.data)
    }

    async function handleSubmit() {
      if (target) {
        // Edit → PUT request
        await api.put(`/admin/atlet/${target.id}`, form)
        // Kalau ada username baru → buat akun login
        if (form.username && form.password) {
          await api.post('/admin/create-account', {
            ref_id: target.id, role: 'atlet',
            username: form.username, password: form.password
          })
        }
      } else {
        // Tambah → POST request (username & password otomatis buat akun)
        await api.post('/admin/atlet', form)
      }
      load() // Refresh data
    }

    async function handleHapus() {
      await api.delete(`/admin/atlet/${target.id}`)
      load()
    }
  }

Fitur menarik di sini: saat admin menambah atlet, ada form
untuk mengisi username dan password. Sehingga atlet tersebut
langsung bisa login ke dashboard-nya tanpa perlu proses terpisah.

[BAGIAN 3 - ROUTE ADMIN DI BACKEND]
Di backend, routes/admin.js menangani semua CRUD:

  // GET semua atlet dengan join ke tabel cabor
  router.get('/atlet', ...guard, async (req, res, next) => {
    try {
      const [rows] = await db.query(`
        SELECT a.*, c.nama AS cabor_nama
        FROM atlet a
        LEFT JOIN cabor c ON a.cabor_id = c.id
        ORDER BY a.nama
      `)
      res.json({ success: true, data: rows })
    } catch (err) { next(err) }
  })

  // POST tambah atlet + buat akun user kalau ada username
  router.post('/atlet', ...guard, async (req, res, next) => {
    try {
      const { nama, cabor_id, username, password, ...rest } = req.body
      const id = uuid()
      await db.query('INSERT INTO atlet (id, nama, cabor_id, ...) VALUES (...)',
        [id, nama, cabor_id, ...])

      // Buat akun login atlet otomatis
      if (username && password) {
        const hash = await bcrypt.hash(password, 10)
        await db.query('INSERT INTO users (id, username, password_hash, role, ref_id) VALUES (...)',
          [uuid(), username, hash, 'atlet', id])
      }
      res.status(201).json({ success: true, id })
    } catch (err) { next(err) }
  })

Dengan menggunakan mysql2/promise, semua query database bisa
ditulis dengan async/await yang bersih dan mudah dibaca.
UUID digunakan sebagai primary key agar ID tidak mudah ditebak.

[BAGIAN 1 - LANDING PAGE SINKRON DATABASE]
Landing page sekarang menampilkan statistik real-time dari database,
bukan lagi data statis. Ini dilakukan dengan useEffect yang
memanggil API saat komponen pertama kali dimuat:

  const [stats, setStats] = useState({
    totalAtlet: 0, totalCabor: 0, totalMedali: 0, medaliEmas: 0,
    caborAktif: []
  })

  useEffect(() => {
    async function loadStats() {
      // Fetch tiga endpoint sekaligus dengan Promise.all
      const [rAtlet, rCabor, rPrestasi] = await Promise.all([
        api.get('/admin/atlet'),
        api.get('/admin/cabor'),
        api.get('/admin/prestasi'),
      ])

      const caborAktif = rCabor.data.data.filter(c => c.status === 'aktif')

      setStats({
        totalAtlet:  rAtlet.data.data.length,
        totalCabor:  caborAktif.length,
        totalMedali: rPrestasi.data.data.length,
        medaliEmas:  rPrestasi.data.data.filter(p => p.medali === 'emas').length,
        caborAktif,
      })
    }
    loadStats()
  }, [])

Promise.all digunakan agar tiga request berjalan paralel,
bukan berurutan, sehingga loading lebih cepat. Jika backend
mati, stats tetap 0 tanpa crash berkat catch yang sudah disiapkan.

Sementara konten publik seperti berita, pengumuman, kegiatan,
dan galeri tetap dikelola lewat Context API dengan localStorage —
karena admin mengedit konten tersebut langsung dari panel admin
frontend, bukan melalui API.
